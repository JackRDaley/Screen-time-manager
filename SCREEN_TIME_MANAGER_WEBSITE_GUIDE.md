# Screen Time Manager Website Guide

This guide defines how the Screen Time Manager marketing website should be designed, structured, and written.

It should be used alongside `SCREEN_TIME_MANAGER_DESIGN_GUIDE.md`, which defines the overall product design system. This website guide applies that system specifically to the public-facing website.

---

## 1. Website Purpose

The website exists to make Screen Time Manager feel real, trustworthy, and easy to understand.

The website should help visitors quickly answer:

1. What is Screen Time Manager?
2. What problem does it solve?
3. How does it work?
4. Is it safe/trustworthy?
5. Where can I install it?

The site should not try to be overly clever, complicated, or flashy. It should act as a clean marketing layer for the Chrome extension.

Primary goal:

> Convert interested visitors into Chrome Web Store installs.

Secondary goals:

> Build trust, explain the product, collect feedback, and show that the extension is actively maintained.

---

## 2. Design Relationship to the Extension

The website should feel like the same product as the Chrome extension.

It should reuse the general Screen Time Manager design language:

- Google-style blue accent
- White cards
- Light gray backgrounds
- Clean sans-serif typography
- Rounded corners
- Subtle borders
- Minimal shadows
- Calm, practical tone

The website should not feel like a separate startup brand or a generic SaaS template.

Important rule:

> Design the website as an extension of the actual Chrome extension UI.

The hero mockup, feature sections, and screenshots should look like they belong to the same product as the extension popup, dashboard, and blocked page.

---

## 3. Pages

The website should be simple.

Required pages:

- `/` — landing page
- `/privacy` — privacy policy / privacy explanation
- `/changelog` — product updates
- `/feedback` — feedback/contact form or instructions

Optional future pages:

- `/features`
- `/pricing`
- `/support`
- `/about`
- `/install`

Do not add unnecessary pages until there is a real need.

---

## 4. Homepage Structure

The homepage should follow this structure:

1. Navigation
2. Hero section
3. Product UI mockup or screenshot
4. Problem section
5. Feature section
6. How it works section
7. Privacy/trust section
8. Changelog/update preview
9. Final call to action
10. Footer

The page should be easy to scan. A visitor should understand the product within 5 seconds.

---

## 5. Navigation

The navigation should be minimal and clean.

### Left Side

Include:

- Product icon/logo
- Product name: `Screen Time Manager`

### Right Side

Include links:

- Features
- Privacy
- Changelog
- Feedback

Include a primary button:

- `Add to Chrome`

### Navigation Style

Use:

- White or translucent background
- Light bottom border
- Clean spacing
- Small product icon
- Sticky nav only if it feels clean and unobtrusive

Avoid:

- Huge navbars
- Too many links
- Dropdown menus
- Startup-style nav clutter

---

## 6. Hero Section

The hero is the most important section on the site.

It should immediately say what the product does.

### Recommended Headline

> Block distracting websites and take back your focus.

Alternative headline options:

> Set limits on distracting websites before they steal your time.

> A simple website blocker and screen time tracker for Chrome.

> Stay focused with simple website limits for Chrome.

### Recommended Subheadline

> Screen Time Manager helps you block distracting sites, set daily limits, and understand where your time goes — right from Chrome.

Alternative subheadline:

> Add the sites that pull you off task, set limits or focus blocks, and let Screen Time Manager enforce them automatically.

### Hero Buttons

Primary CTA:

> Add to Chrome

Secondary CTA:

> See how it works

or

> View features

The primary CTA should always be more visually prominent.

### Hero Visual

The hero should include a product-focused visual, not an abstract illustration.

Use one of these:

- Chrome extension popup mockup
- Dashboard mockup
- Blocked page mockup
- Browser window containing the app UI
- Combined mockup showing dashboard + blocked page

The visual should show realistic product content such as:

- `youtube.com blocked`
- `Daily limit reached`
- `25 blocked page views today`
- `Focus block active`
- `Top distracting sites`
- `Add domain`
- `Daily usage`

The mockup should look realistic enough to communicate the product, but polished enough for marketing.

---

## 7. Problem Section

The problem section should be short and relatable.

It should explain the user pain without sounding dramatic.

Possible copy:

> You open YouTube for one video. You check Reddit for a minute. You glance at social media between assignments. Suddenly, your focus is gone and 45 minutes disappeared.

Then transition:

> Screen Time Manager adds the friction you need before distractions turn into habits.

Keep this section concise. Do not make it overly emotional or preachy.

---

## 8. Features Section

Use 4–6 clean feature cards.

Each card should include:

- Simple line icon
- Short heading
- One or two sentence description

Recommended feature cards:

### Website Blocking

Block distracting sites before they pull you back in.

### Daily Limits

Set time limits for specific domains so quick checks do not turn into long sessions.

### Scheduled Focus Blocks

Create time blocks where distracting websites stay unavailable.

### Activity Dashboard

See blocked pages, added domains, and usage patterns in one simple view.

### Simple Domain Management

Add, remove, and manage distracting websites without a complicated setup.

### Privacy-Conscious Tracking

Track only what is needed to provide blocking, limits, and basic usage stats.

### Feature Card Style

Use:

- White background
- Light border
- Rounded corners
- Subtle shadow if needed
- Blue/gray line icons
- Clear headings
- Calm body text

Avoid:

- Huge feature cards with too much text
- Overly colorful icons
- Generic SaaS claims
- Features the extension does not actually have

---

## 9. How It Works Section

This section should be simple and concrete.

Use three steps:

### 1. Add distracting websites

Add sites like YouTube, Reddit, TikTok, Instagram, X, or any domain that pulls you off task.

### 2. Set limits or focus blocks

Choose daily limits, scheduled blocks, or stricter focus rules.

### 3. Stay focused

Screen Time Manager blocks distractions and helps you see how often you try to return.

Recommended layout:

- Three horizontal cards on desktop
- Stacked cards on mobile
- Numbered steps
- Small icon for each step

---

## 11. Privacy / Trust Section

The privacy section is important because the extension deals with browsing activity.

Keep the language plain and reassuring.

Recommended heading:

> Built to be simple and privacy-conscious.

Recommended copy:

> Screen Time Manager only tracks the activity needed to provide website blocking, limits, and usage stats. The product should feel transparent, lightweight, and under the user’s control.

Possible trust bullets:

- No unnecessary tracking
- No selling user data
- Clear domain-based controls
- Simple privacy policy
- Built for personal focus, not surveillance

Avoid:

- Overpromising
- Vague privacy claims
- Legal-heavy wording on the homepage
- Saying things that are not technically true

If a claim is made on the website, the extension must actually support it.

---

## 12. Changelog / Updates Section

Include a small section showing that the product is active.

Recommended heading:

> Actively improving.

Recommended copy:

> Screen Time Manager is being updated with better blocking tools, clearer activity stats, and a cleaner focus experience.

Show 2–3 recent update cards if available.

Example:

### Recent updates

- Improved activity dashboard
- Added clearer blocked-page tracking
- Refined domain management flow

Include link:

> View changelog

This makes the product feel maintained and alive.

---

## 13. Feedback Section

The site should make feedback easy.

Recommended heading:

> Help shape Screen Time Manager.

Recommended copy:

> Found a bug, want a feature, or have an idea for making the extension better? Send feedback and help improve the product.

Include:

- Feedback form
- Email link
- GitHub issue link, if public
- Simple contact method

Keep it low-friction. Early users are valuable.

---

## 14. Final Call to Action

End the homepage with a clear CTA.

Recommended heading:

> Ready to take back your focus?

Recommended subheading:

> Add Screen Time Manager to Chrome and start blocking the sites that pull you off task.

Primary button:

> Add to Chrome

Secondary link:

> Read privacy policy

The final CTA should be simple and confident.

---

## 15. Footer

The footer should be minimal.

Include:

- Product name
- Short description
- Chrome Web Store link
- Privacy
- Changelog
- Feedback/contact
- Optional GitHub link

Example footer description:

> A simple Chrome extension for blocking distracting websites, setting limits, and understanding your browsing habits.

Avoid a giant footer with unnecessary columns.

---

## 16. Copywriting Rules

The website copy should be clear, direct, and practical.

### Good Copy

- `Block distracting websites before they pull you back in.`
- `Set limits once. Let Screen Time Manager enforce them.`
- `See where your attention is going.`
- `Built for students, builders, and anyone trying to stay focused.`
- `Simple website limits for Chrome.`

### Avoid

- `Unlock your ultimate productivity potential.`
- `Revolutionize your workflow.`
- `Crush your goals with next-gen focus technology.`
- `Transform your digital wellness journey.`
- `AI-powered productivity optimization.`

The product should sound useful, not hype-heavy.

---

The website should use:

- Light backgrounds
- White cards
- Blue accents
- Dark readable text
- Clean spacing
- Rounded corners
- Subtle borders
- Simple icons
- Realistic product mockups

Avoid:

- Purple/blue AI gradients
- Dark crypto-style layouts
- Neon glow effects
- Stock photos
- Generic 3D illustrations
- Excessive animation
- Overly complex charts
- Cluttered sections

The design should support focus, not compete with it.

## 20. Technical Implementation Notes

Recommended stack:

- Next.js or Vite/React
- Tailwind CSS
- TypeScript if the repo already uses it
- Lucide icons
- Vercel deployment

Implementation rules:

- Keep components small and readable
- Use semantic HTML
- Make the site responsive
- Use accessible buttons and links
- Use real CTA links where available
- Do not invent fake features
- Do not add unnecessary dependencies
- Do not overanimate
- Keep load time fast

## 23. Mobile Requirements

Mobile layout should be clean and readable.

Rules:

- Stack feature cards
- Keep hero text clear
- Put CTA near the top
- Do not make mockups too tiny
- Use large tap targets
- Avoid horizontal scrolling
- Keep nav simple
- Use a mobile menu only if necessary

The mobile site should still communicate the product within a few seconds.

---

## 24. Website Success Criteria

The website is successful if a first-time visitor can quickly understand:

1. This is a Chrome extension.
2. It blocks distracting websites.
3. It can set limits and focus blocks.
4. It includes basic activity tracking.
5. It is privacy-conscious.
6. It is actively maintained.
7. They can install it from the Chrome Web Store.

The site should make the product feel more legitimate, not more complicated.

---

## 25. Agentic Model Instructions

When using an AI coding agent to work on the website, include these instructions:

> Use `SCREEN_TIME_MANAGER_DESIGN_GUIDE.md` as the source of truth for the brand and visual system.
>
> Build the website as a clean marketing layer for the Chrome extension, not a generic SaaS landing page.
>
> The site should look visually connected to the actual extension UI: white cards, blue accents, light gray backgrounds, rounded corners, subtle borders, clean typography, and realistic product mockups.
>
> Do not invent features that do not exist.
>
> Prioritize clarity, trust, and install conversion.
>
> Keep the site simple, responsive, accessible, and easy to maintain.

---

## 26. Final Direction

The website should make Screen Time Manager feel like a polished, useful Chrome extension built by someone who is actively improving it.

It should not oversell.

It should not look generic.

It should not distract from the product.

It should help users think:

> This is simple, trustworthy, and probably worth installing.
